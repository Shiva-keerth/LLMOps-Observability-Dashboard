A production-grade, real-time observability dashboard engineered to monitor autonomous LLM agents and evaluation pipelines. This dashboard connects directly to SQLite telemetry databases to track critical metrics like token consumption, inference latency, multi-model fallback rates, and deterministic pre-filter rejection funnels.

> **Designed for the Jobline Pipeline:** This specific dashboard monitors the autonomous AI job scraper, providing full transparency into how the Agentic Evaluators are performing in real-time.

---

## 🚀 Key Features

*   📈 **Real-Time KPIs:** Tracks Total Jobs Evaluated, Average API Latency, Total Tokens Consumed, and computes estimated API costs (based on Llama-3 70B pricing).
*   📉 **Funnel Analytics:** Visualizes the exact drop-off points of data passing through the system:
    *   *Scraped ➔ Passed Blacklist ➔ Passed Regex Pre-filter ➔ LLM Accepted*
*   🔄 **Model Fallback Tracking:** Monitors multi-model fallback chains (e.g., GPT OSS -> Qwen 2.5) to surface reliability metrics and API uptime.
*   ⚠️ **Error Diagnostics:** Logs total chain exhaustion events and surfaces the top regex disqualification strings, ensuring zero silent failures in production.
*   🔒 **Data Privacy Ready:** Built to read anonymized telemetry databases (mapping to "Company A", "Company B") for secure public demonstration and compliance.

---

## 📸 Dashboard Interface

The dashboard is built with a wide-layout Streamlit interface, separating high-level business metrics from deep technical diagnostics.

*   **KPI Bar:** Instant visibility into latency and token costs.
*   **Funnel Chart:** Visualizes how much data the LLM rejected vs. how much was killed early by deterministic Regex filters (saving token costs).
*   **Raw Telemetry Feed:** A searchable, filterable data grid of the raw underlying event logs.

## 🛠️ Technology Stack

*   **Frontend:** Streamlit, Pandas (Data Manipulation), Streamlit Native Charts
*   **Backend / DB:** SQLite3 (Telemetry Storage)

## 💻 Local Development Setup

To run this dashboard locally, ensure you have the `demo_telemetry.db` SQLite database in the root directory (generated by the Jobline Pipeline).

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/Shiva-keerth/LLMOps-Observability-Dashboard.git
    cd LLMOps-Observability-Dashboard
    ```

2.  **Create a Virtual Environment & Install Dependencies:**
    ```bash
    python -m venv .venv
    .\.venv\Scripts\activate      # Windows
    pip install -r requirements.txt
    ```

3.  **Run the Streamlit App:**
    ```bash
    streamlit run app.py
    ```

## 🐳 Docker Deployment

To deploy this in a containerized production environment:

```bash
docker build -t llmops-dashboard .
docker run -p 8501:8501 llmops-dashboard
```
